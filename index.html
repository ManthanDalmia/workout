Key notes ▾

Zone 2 runs

Conversational pace, 20 min max on weekdays. Warms you up without eating into recovery.

Long run — Sunday

70% effort, 8–12 km. Aerobic base, not speed.

Progressive overload

Add a little weight or a rep on the main lifts weekly. 1–2 kg/week compounds fast.

Protein

150–165 g/day (2–2.2 g/kg). Key for holding muscle in a deficit.

Rest between sets

2–3 min on compounds, 60–90 s on accessories.

Mon + Thu rest

Two mid-week rest days = fresh sessions. Recovery is training.

Calorie deficit

Keep it small, 200–300 kcal. Too aggressive costs muscle.

Review point

Run this 4-day split 2–3 weeks, then move to a 5-day split.

Host it & turn on cloud sync ▾

By default this saves on the device you open it on. To get it online and syncing across phone + laptop:

1 · Host on GitHub Pages
  1. Create a public GitHub repo and add this file as index.html.
  2. Repo Settings → Pages, set source to main / root.
  3. It goes live at https://<you>.github.io/<repo>/.
2 · Add Firebase (Firestore + Google sign-in)
  1. In the Firebase console, create a project, then Firestore Database → Create.
  2. Authentication → Get started → Sign-in method, enable Google, and set a support email.
  3. Authentication → Settings → Authorized domains: add your Pages domain, e.g. <you>.github.io.
  4. Project settings → Your apps → Web (</>) and copy the firebaseConfig object.
  5. Paste it into FIREBASE_CONFIG near the top of the script in this file, then re-deploy.
3 · Lock the data to each account

In Firestore → Rules, paste this so people can only read/write their own workouts:

rules_version = '2';
service cloud.firestore {
  match /databases/{db}/documents {
    match /users/{uid}/workouts/{doc} {
      allow read, write: if request.auth != null
                         && request.auth.uid == uid;
    }
  }
}

Once hosted with a config, you'll get a Continue with Google screen; after signing in the badge shows ☁ Synced and your log follows you across devices.

Note: Firebase only works on the hosted page, not a local file. Opened locally it falls back to on-device storage with no sign-in. Your data lives under users/<your-uid>/workouts, private to your Google account.
